Key Changes in v2.0
1
The older version required a separate deployment for each pipeline because it filtered stats based on server name.
2
The newer version handles all flows in a single deployment by routing based on filename (CCF Nodes) or Route value instead.
3
StatsCollector stores all stats in one common directory.
4
The pipeline is identified by the CCF Node or Route value in the filename, and the corresponding report is generated accordingly.
